# Praxia ビジネス発展計画

> **このドキュメントの位置付け**: OSS としての Praxia を公開・拡散した後、どのように事業として発展させていくかの行動計画書。
> 想定読者: ご自身、共同創業者候補、初期エンジェル投資家、相談相手のメンター。

---

## 0. エグゼクティブ・サマリ

| 項目 | 内容 |
|------|------|
| **製品** | Praxia — 業務特化型マルチエージェント・オーケストレーター + 個人→組織メモリ自動循環機構 |
| **OSS コア** | Apache 2.0、6 業務スキル + 3 特化フロー + 5 層メモリ + 認証 |
| **ターゲット市場** | 日本国内の大企業・中堅企業のナレッジワーク部門 (営業 / 法務 / 知財 / 設計 / 購買 / 投資) |
| **収益モデル** | オープンコア + コンサル + Vertical SaaS の 3 経路 |
| **目標** | 24 ヶ月で ARR 1 億円、36 ヶ月で 5 億円、シリーズ A 調達ライン到達 |

3 行で言うと:
> 「個人で使うだけで暗黙知が組織知として育つ AI フレームワーク」を OSS で広く普及させ、
> 企業導入時に必要となる **エンタープライズ拡張・業界別ベスプラ・実装支援** で収益化する。
> Open Core 戦略で、技術力 (OSS) と収益 (有償拡張) を両輪で回す。

## 1. 市場機会の分析

### 1.1 マクロトレンド (2026 年時点)

- **AI エージェント導入の臨界点**: 2024〜2025 年に PoC 段階を脱し、本番投入が加速
- **「LLM 民主化」の終焉**: モデル性能の差別化はほぼ終了、勝負は **「現場での効果」**
- **属人化の限界**: 大企業の AI 活用度は「個人の最高水準の 30〜50%」で頭打ち
- **オープンソース + 有償ソリューション** のハイブリッドが Github 等で主流に

### 1.2 ターゲット市場のサイズ感

日本国内のナレッジワーク AI 関連市場:

| セグメント | 市場規模 (2026 推定) | 年成長率 | Praxia 適合度 |
|-----------|--------------------|---------|--------------|
| エンタープライズ AI プラットフォーム | 約 800 億円 | +35%/年 | ★★★ |
| 業務特化型 SaaS (Vertical AI) | 約 400 億円 | +50%/年 | ★★★ |
| ナレッジマネジメント | 約 200 億円 | +20%/年 | ★★ |
| AI コンサルティング | 約 1,200 億円 | +25%/年 | ★★★ |
| **合計** | **約 2,600 億円** | | |

Praxia がリーチ可能な対応市場 (SAM): 約 200〜400 億円規模。

### 1.3 競合分析

| カテゴリ | 代表プレイヤー | Praxia との差別化 |
|---------|--------------|-----------------|
| 汎用エージェント FW | LangGraph, CrewAI, AutoGen | **業務フロー特化** + 個人→組織循環 |
| Vertical AI | Harvey (法務), Glean (検索) | **OSS** + 自社運用 + マルチドメイン |
| メモリプラットフォーム | Mem0, Letta, Zep | **オーケストレーターと一体** + ビジネススキル同梱 |
| エンタープライズ KM | Confluence, Notion AI | **能動学習** + マルチエージェント |
| 国内 SIer 製品 | 富士通, NTT, NEC の AI 製品 | **OSS による速度・透明性** + 国際展開可能性 |

**勝ち筋**: 業務特化テンプレート + 個人→組織メモリ循環 という独自の組合せは現在誰も提供していない。

## 2. 収益モデルの設計 (オープンコア・モデル)

### 2.1 3 つの収益経路

```
                Praxia OSS (Apache 2.0)
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
     経路A        経路B        経路C
   コンサル    エンタープライズ   Vertical SaaS
                  版                   
   個別実装      管理機能・        業種特化型
   PoC支援      SSO・監査         月額 SaaS
   研修         プレミアム
   
   早期収益    継続的収益       高粗利益
   高単価      予測可能         スケール可能
```

### 2.2 経路 A: コンサルティング・実装支援 (Phase 1: 0-12 ヶ月)

**ターゲット**: OSS を見て興味を持った大企業・中堅企業 (10〜100 名規模の AI 推進部門)

**サービス内容**:
- **PoC 支援パック (300〜500 万円 / 案件)**: 1 業務に絞った 3 ヶ月 PoC、効果測定込み
- **本番導入支援 (800〜1,500 万円 / 案件)**: 6 ヶ月、全社展開設計 + 業種別カスタマイズ
- **トレーニング (50〜100 万円 / 回)**: 半日〜1日のオンサイト研修
- **継続サポート (月額 30〜50 万円)**: 改善・追加機能・相談対応

**収益見通し**:
| 期間 | 案件数 | 平均単価 | 累積売上 |
|------|-------|---------|---------|
| 0-6ヶ月 | 2-3 件 | 400 万円 | 800-1,200 万円 |
| 6-12ヶ月 | 4-6 件 | 600 万円 | 3,200-4,800 万円 |
| 12-24ヶ月 | 8-12 件 | 800 万円 | 1.0-1.6 億円 |

**獲得チャネル**: Zenn / X / GitHub Star → Inbound 問い合わせ。RAG-1 グランプリ実績で権威性確立。

### 2.3 経路 B: エンタープライズ版・マネージド (Phase 2: 12-24 ヶ月)

**ターゲット**: コンサル経由で導入した企業の継続契約、および大手 SaaS 企業

**プロダクト構成**:

| エディション | 月額 | 含まれる機能 | ユーザ規模 |
|-------------|------|------------|----------|
| **Free** (OSS) | ¥0 | 全 OSS 機能 | 〜 |
| **Team** | ¥30k〜100k | 共有メモリのチーム機能、基本ダッシュボード | 5〜30 名 |
| **Business** | ¥150k〜500k | SSO (OIDC/SAML), 詳細監査, 多テナント | 30〜500 名 |
| **Enterprise** | ¥800k 〜 (要相談) | オンプレ展開, 専任 SE, SLA, カスタマイズ | 500 名〜 |

**有償機能の例** (OSS にはない):
- マルチテナント・GUI 管理画面
- SSO/OIDC/SAML 統合
- きめ細かい RBAC 設定 (カスタムロール)
- SOC2 / ISO27001 準拠の詳細監査
- 高可用性・冗長構成
- 専任カスタマーサクセス
- SLA (99.9% / 24h レスポンス)

**ターゲット規模**:
| 期間 | 顧客数 | 平均 ARPU | ARR |
|------|-------|----------|-----|
| 12-18ヶ月 | 5-10 社 | 月 30 万円 | 1,800-3,600 万円 |
| 18-24ヶ月 | 15-30 社 | 月 50 万円 | 9,000 万-1.8 億円 |
| 24-36ヶ月 | 50-100 社 | 月 80 万円 | 4.8-9.6 億円 |

### 2.4 経路 C: Vertical SaaS — 業界特化版 (Phase 3: 24 ヶ月以降)

**コンセプト**: OSS ベースに **業界特有の RAG ナレッジ + 高精度プロンプト** を上載せした完全パッケージ。

**第一弾候補 (優先順)**:

1. **Praxia Sales** — B2B 営業特化版 (業界 IR, 競合 DB, CRM 連携付き) — ¥10-30k/seat/月
2. **Praxia Legal** — 中堅企業の法務部向け (国内法 DB, 過去判例, 契約テンプレ付き) — ¥30-80k/seat/月
3. **Praxia Patent** — 知財部門向け (J-PlatPat 連携, 特許 DB アクセス) — ¥50-150k/seat/月
4. **Praxia R&D** — 製造業 R&D 向け (技術文献, 特許マップ, 設計支援) — ¥80-200k/seat/月

**Vertical SaaS の収益力**:
- 100 seat × ¥30k × 10 社 = 月 3,000 万円 = **ARR 3.6 億円 / 製品**
- 4 製品成功で **ARR 14 億円超** が見える

## 4. Go-To-Market 戦略

### 4.1 マーケティング・ファネル

```
        ┌────────────────────────────────┐
        │  認知 (Awareness)              │
        │  Zenn / X / GitHub Star        │
        │  RAG-1 実績で権威性            │
        └─────────────┬──────────────────┘
                      │
        ┌─────────────▼──────────────────┐
        │  関心 (Interest)                │
        │  README + ユースケース         │
        │  Quickstart で 5 分で動く      │
        └─────────────┬──────────────────┘
                      │
        ┌─────────────▼──────────────────┐
        │  検討 (Consideration)           │
        │  業種別事例 + 比較表           │
        │  PoC 提案 (300-500 万円)       │
        └─────────────┬──────────────────┘
                      │
        ┌─────────────▼──────────────────┐
        │  契約 (Decision)                │
        │  PoC で効果実証 → 本番化       │
        │  → サブスク継続契約            │
        └────────────────────────────────┘
```

### 4.2 Tier 別の獲得アプローチ

| 顧客 Tier | 獲得方法 | 営業サイクル | 平均単価 |
|----------|---------|-------------|---------|
| Tier 1: 大企業 (1,000 名+) | 紹介 + カンファレンス | 6-12 ヶ月 | 1,000-3,000 万円/年 |
| Tier 2: 中堅 (100-1,000 名) | Inbound (Zenn/SEO) | 2-4 ヶ月 | 300-800 万円/年 |
| Tier 3: 小規模・スタートアップ | OSS Self-serve → SaaS | 数日〜1 ヶ月 | 月 3-30 万円 |

### 4.3 採用フィルター

「導入を検討すべき企業」の条件:
- ✅ ナレッジワーク部門が 30 名以上
- ✅ 既存の AI/LLM ツール (ChatGPT Enterprise, Copilot 等) を導入済み
- ✅ ベテラン社員の引退・転職リスクが顕在化
- ✅ AI 活用度のばらつきが社内で 2 倍以上

3 つ以上 ✅ なら、Praxia の導入効果が出やすい体質。

## 7. リスクと対策

| リスク | 影響度 | 対策 |
|-------|-------|------|
| 大手 (OpenAI, Anthropic) が同等機能をリリース | 高 | OSS の機動性 + 業界特化で差別化、提携先模索 |

| 規制 (個人情報・AI 規制) | 中 | プライバシー設計を OSS から徹底、コンプラ専門家を顧問に |
| ベンダーロックイン批判 | 低 | プラガブル設計を維持、移行ツールを提供 |

## 10. 創業者としてのコミットメント

**最初の 12 ヶ月のフルタイム投入の覚悟**:
- 副業 / 二足のわらじではなく、可能な限り早期に独立
- 自己資金で 6-12 ヶ月を凌ぐ覚悟、または初期エンジェル調達

**価値観**:
- **OSS への忠実さ** — Apache 2.0 のコアは絶対に閉じない
- **コミュニティ・ファースト** — 用心深く、丁寧に、長期目線で
- **エビデンス主義** — 数値で語る、プロダクト・データで判断する

**「成功」の定義**:
- 5 年後: グローバルで Praxia が「マルチエージェント・メモリ」のデファクトに
- 個人として: 「個人の天才性 × 組織の継続性」を AI で結ぶことに人生を賭けたと胸を張れる状態
