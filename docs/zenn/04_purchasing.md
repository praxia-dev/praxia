## 前提となる現場の課題

製造業の購買部門の典型的な失敗:

> 「単価が安いという理由でサプライヤーを選定したが、実は児童労働懸念で取引先が西欧顧客から監査を受け、最終的にシングルソース解消で 2 億円の追加コスト」

直接コストだけ見ると見落とすリスクが多すぎる。一方で全観点を一律にチェックする工数もない。

## ユースケース 3 つ

### UC-1: 新規サプライヤー候補の初期評価

PurchasingSkill には **QCD+S フレームワーク** が組み込まれています:

- **Q**uality: 品質保証体制・実績・認証 (ISO 9001 等)
- **C**ost: 直接コスト + 物流 + 関税 + 在庫 + 不良率を含む TCO
- **D**elivery: リードタイム・納期遵守率・柔軟性
- **S**ustainability: ESG / カーボンフットプリント / 児童労働排除

```bash
praxia skill run purchasing "
新規サプライヤー候補 30 社を 5 社にショート化:

要件:
- 部品: 高密度実装 PCB
- 年間調達量: 200 万枚
- 当社所在: 日本 (関東)
- 納期要求: リードタイム 4 週間以内
- 認証: ISO 9001、IATF 16949 (車載部品向け)
- 制約: ロシア・ベラルーシ関連企業との取引なし

各候補について QCD+S 全観点で評価し、上位 5 社を理由付きで推薦してください。
"
```

**出力例**:

### UC-2: RFQ 回収結果の TCO 比較

設備投資 5 億円規模の見積依頼に対して 5〜8 社から RFQ を回収。

```python
from praxia.skills import PurchasingSkill

skill = PurchasingSkill()

# RFQ 結果を構造化データで渡す
rfq_results = """
社名 / 単価 / リードタイム / 不良率 / 物流費 / 関税 / 在庫保有義務 / 為替条件
A 社 / 1,200 円 / 28 日 / 0.5% / 50 円 / 0% / 1 ヶ月 / JPY
B 社 / 1,050 円 / 35 日 / 1.2% / 80 円 / 5% / 2 ヶ月 / USD
C 社 / 1,300 円 / 21 日 / 0.3% / 30 円 / 0% / 0.5 ヶ月 / JPY
D 社 / 1,100 円 / 30 日 / 0.8% / 60 円 / 3% / 1 ヶ月 / EUR
E 社 / 1,250 円 / 25 日 / 0.4% / 40 円 / 0% / 0.5 ヶ月 / JPY
"""

tco_analysis = skill.run(f"""
以下の RFQ 結果について、年間調達量 200 万枚での TCO を計算し、
真の最安サプライヤーを特定してください。

{rfq_results}

考慮すべきコスト:
- 直接コスト (単価 × 量)
- 物流コスト
- 関税
- 不良率による廃棄コスト
- 在庫保有コスト (倉庫費 + 機会損失)
- 為替変動リスク (3 ヶ月分のヘッジコスト推定)
- リードタイム延長による在庫増加リスク

出力: 年間 TCO 比較表 + 推奨選定 + 経営層向けサマリ
""")
```

**結果のポイント**: 単価最安の B 社が、TCO で見ると 4 番目に。為替・関税・不良率を加味すると A 社や E 社の方が **15〜20% 安い**。

| 観点 | Before | After |
|------|--------|-------|
| TCO 試算所要時間 | 1〜2 週間 | **2〜3 営業日** |
| 隠れコストの織り込み | 直接コスト中心 (+ 物流) | **直接 + 物流 + 関税 + 在庫 + 不良率 + 為替** |
| ベンダーロックイン評価 | 担当者の感覚 | **定量的なリスクスコア** |
| 経営層への説明資料 | 別途 1 週間 | **同時生成** |

**訴求ポイント**: 「TCO 視点で初期見積より 30% 高い真のコストを発見できる」 → 経営層への説得材料

### UC-3: BCP 観点のサプライチェーン棚卸し

コロナ・ウクライナ・台湾海峡リスクを受けた、全社サプライヤー 500 社の BCP 棚卸し。

```python
from praxia import Praxia
from praxia.skills import PurchasingSkill
import pandas as pd

# 全サプライヤー DB を読み込み
suppliers = pd.read_csv("all_suppliers.csv")
p = Praxia(user_id="bcp_team", default_model="claude")

results = []
for _, row in suppliers.iterrows():
    skill = PurchasingSkill()
    risk = skill.run(f"""
    BCP リスク評価:

    サプライヤー: {row['name']}
    所在地: {row['country']}, {row['city']}
    部品: {row['part_type']}
    年間取引額: {row['annual_volume']} JPY
    シングルソース: {row['is_single_source']}
    代替候補数: {row['alternatives']}

    以下を評価:
    1. 地政学リスク (現地点でのリスクレベル)
    2. 自然災害リスク
    3. 経営リスク (財務悪化・倒産可能性)
    4. シングルソース解消の困難度
    5. 総合 BCP リスクスコア (0-100)

    重大度別の推奨アクションも提示。
    """)
    results.append({"supplier": row['name'], "risk": risk})

# 全 500 社の評価を集計、ダッシュボード化
```

| 観点 | Before | After |
|------|--------|-------|
| 500 社の評価所要時間 | 6 ヶ月 (専任チーム 5 名) | **6〜8 週間** |
| シングルソース検出率 | 70〜80% | **95% 以上** |
| リスク・マトリクスの精度 | 国レベル | **県・都市レベル + サプライヤー固有** |
| 経営報告までの期間 | 半年〜1 年 | **2 ヶ月** |

## ガードレール — 下請法・独禁法の自動チェック

PurchasingSkill には日本の購買特有のガードレールが組み込まれています:

```python
class PurchasingSkill(Skill):
    system_prompt = """
    ...
    【ガードレール】
    - 下請法 (60日以内支払・買い叩き禁止) の遵守を必ずチェック
    - 独占禁止法に抵触する優越的地位の濫用を避ける
    - サプライヤーの財務データは公開範囲のみ参照
    ...
    """
```

これにより、RFQ や交渉時に法令違反を未然に防げます。

```python
# 退職前 3 ヶ月で集中利用
```

## 自社カスタマイズ — IFRS 16 リース判定スキル

```python
from praxia.skills.skill import Skill, SkillManifest

class IFRS16LeaseSkill(Skill):
    manifest = SkillManifest(
        name="ifrs16_lease",
        description="IFRS 16 適用判定 (リース vs サービス契約)",
        domain="purchasing",
        tags=["accounting", "ifrs"],
    )
    system_prompt = """
    あなたは IFRS 16 リース判定の専門家です。

    【役割】
    契約書を読み、IFRS 16 におけるリース要件を判定する:
    1. 識別された資産 (Identified Asset) があるか
    2. 経済的便益のほぼ全てを得る権利 (Right to Obtain Economic Benefits)
    3. 使用方法を指示する権利 (Right to Direct the Use)

    全 3 要件を満たせば「リース」、いずれか欠けば「サービス契約」と判定。

    【ガードレール】
    - 最終判定は会計士の確認必須を明記
    - 監査法人見解との不整合可能性を提示
    """
```
